嘿，我要不要开家咖啡馆

发表于 2017-06-15 | 分类于 产品思维

“嘿，我要不要在写字楼下开家咖啡馆”。这是前两天看到的一个产品经理面试题。我猜要考的应该是应聘者的产品思维，作为入行已经3个多月的PM小白一枚，斗胆试着用做产品的思路来答一发。

  这个问题听着有点耳熟，我平时处理的多是项目上的需求，经常会有项目人员来问，“Hi，zhangmin，我们要不要加上Y功能啊？”其实对于这个问题，我认为要考量的有一下几点：

  1. 探寻用户想做这个功能背后的真实意图；
  2. 用户的现状与预期；
  3. 该功能是否满足用户的真实需求 ；
  4. 该功能是否能达到用户的预期。

挖掘用户需求

嘿，哥们，你为啥想在写字楼下开咖啡馆啊

  PM重中之重是找准用户的真正需求。在工作中，项目人员、解决方案人员给我们提的需求很多时候已经脱离用户的真实意图了。怎么理解呢，举个例子，用户需求：“我想在客厅墙壁上挂副画”。项目技术支持人员的理解：“客户想在墙壁上钻个洞”。产品人员接受到来自项目人员提交上来的需求后，给出的方案是一个电钻。但其实用户只是想要一个锤子。这是需求传递失真的问题。项目人员知道客户的功能需求，但是在不清楚功能需求(包括应用场景)和技术需求对应关系的情况下，向产品经理传达了被曲解了的解决方案。而产品经理，在接到要求后，并没有了解客户为什么有这样的需求，是为了解决什么问题、还是为了达到什么样的更好的效果。所以，产品经理一定要先找准用户的需求再下手。

喜欢的妹子刚好在这栋楼里上班，为了追求妹子开咖啡馆。
为了挣钱，自主创业。

上面的三种回答，是我瞎猜、随便写的。结合用户的回答，仅仅针对开不开咖啡馆这个问题，就有不同的处理方式。
第一种：用户的需求是追求妹子，开咖啡馆是他追求妹子的一种方案。用户心中，吸引妹子>盈利。
第二种：以挣钱为目的。用户心中盈利与成本控制是放在在第一位。

用户愿景–明确开不开咖啡馆的标准

  上一个步骤，通过与客户交谈，明确了用户的需求–开咖啡馆。接下来是确立产品目标，用户开不开咖啡馆的标准是什么。例如：想追求妹子的哥们认为，可以开一家咖啡馆的标准是：每天能见到心仪的妹子，赚不赚钱不管。但想赚钱的哥们肯定想，我开咖啡馆要能做到月均纯利润2W以上才行。

  用户的预期不同，制定方案时考量的标准也不同。针对想追求妹子的哥们用户来说，我们产品人员制定的方案应该偏重考虑妹子每天上下班会不会从这儿经过、妹子喜不喜欢喝咖啡、如何吸引心仪的妹子……而对于以挣钱为目的的哥们来说，方案的核心点应该是围绕是否能达到盈利目标。

假设，客户的需求是第二种。接下来我们要做的事情就是围绕客户的需求与预期，制定方案。

头脑风暴所有与咖啡馆相关的名称、搜集资料

  对于自己不太熟悉的领域，我会先用思维导图进行一番头脑风暴，将所有能想到的点罗列出来，再慢慢细化。

根据头脑风暴里提到的考虑点，搜集资料，制定方案，提供给用户做判断参考。

以上，笔拙请见谅。
